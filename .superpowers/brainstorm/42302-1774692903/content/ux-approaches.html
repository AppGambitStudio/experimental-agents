UX Approach for Agent Interfaces

How should non-technical users interact with our agents?

A

Standalone Web App (Next.js / Vite)

Full web application with forms, chat UI, file upload, and result display. Hosted separately (Vercel/AWS). Each agent gets its own page or app.

Pros

  • Best UX for non-technical users
  • Drag-and-drop file upload
  • Real-time progress indicators
  • Shareable URLs for results
  • Mobile-friendly

Cons

  • Needs hosting infrastructure (Vercel/AWS)
  • Separate frontend + backend deployment
  • Most development effort
  • Auth, sessions, storage to manage
B

Claude Code Desktop / Web (claude.ai/code)

Use Claude Code's built-in UI as the interface. Users open Claude Code, the agent runs as a project with skills and slash commands. No custom UI needed — Claude IS the UI.

Pros

  • Zero UI development — already built
  • Chat interface users already know
  • Skills auto-invoke, slash commands work
  • File handling built in
  • Desktop + web + IDE versions exist

Cons

  • Requires Claude Code installation or claude.ai account
  • Less control over UX flow
  • Can't deeply customize the interface
  • User needs to understand "project" concept
C

WhatsApp / Telegram Bot

Users interact via messaging apps they already use. Send a contract PDF on WhatsApp, get back risk analysis. Send property details, get due diligence report.

Pros

  • Zero learning curve — everyone knows WhatsApp
  • Mobile-first (perfect for property buyers in India)
  • File sharing built in (PDFs, photos)
  • Notification for long-running analyses
  • Shareable — forward report to family/lawyer

Cons

  • Limited rich formatting (no tables, charts)
  • WhatsApp Business API has costs and approval process
  • Message size limits (especially for detailed reports)
  • Harder to do multi-step interactive flows
D

Hybrid: Minimal Web Form + Claude Code Backend

Simple single-page web form for input (property details or contract upload). Form submits to Claude Agent SDK running on the server. Results displayed as a clean web report. No chat UI — just input → processing → output.

Pros

  • Simple, focused UX — form + report, nothing else
  • Low development effort (single HTML page + API endpoint)
  • Agent SDK runs server-side — no client complexity
  • Shareable report URLs
  • Can add chat later as enhancement

Cons

  • No interactive back-and-forth (one-shot only initially)
  • Still needs a server (but simpler than full app)
  • Less engaging than chat interfaces
